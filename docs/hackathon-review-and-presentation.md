# PurrScription — обзор проекта и план презентации

Сравнение с методичкой хакатона «Сервис аудио-/видеоразметки Gecko Next».

---

## Что за проект

**PurrScription** — веб-платформа для аудио-/видеоразметки речевых данных (аналог Gecko с расширениями). Рабочее название своё; по ТЗ это допустимо.

**Стек:**

- **Frontend:** React 18 + TypeScript, Vite, Zustand, TanStack Query, WaveSurfer.js v7
- **Backend:** Python 3.12 + FastAPI, SQLAlchemy, Alembic
- **Инфра:** Docker Compose, PostgreSQL, Redis, WebSocket

**Суть:** разметчики исправляют предразметку ASR, верификаторы проверяют, система контролирует качество и экспортирует JSON для ML-пайплайнов.

**Роли (6 штук):** admin, supervisor, annotator, verifier, ml_engineer, customer — с разграничением прав через RBAC.

### Основные экраны

| Экран | Назначение |
|-------|------------|
| Вход / регистрация | Авторизация по JWT |
| Проекты | Создание и список проектов |
| Задачи | Импорт медиа, назначение, статусы |
| Рабочее место | Waveform, видео, сегменты, текст, качество |
| Термины | Словарь спорных терминов |
| Аналитика | Статистика по задачам (клиентская) |
| Админка | Пользователи и роли |

## Всё ли выполнено?

**Нет, не всё** — полное ТЗ на 16 страниц не закрыто.

**Для хакатона — да, базовый сценарий закрыт:**

> загрузка → разметка → проверка → возврат/приёмка → экспорт

Это главное требование из разделов 1.4 и 16 методички. Оценка по критериям приёмки (раздел 14): **~70–80%** при честной демонстрации с указанием ограничений.

### Сильные стороны для жюри

- Полный workflow разметчик → верификатор
- Waveform-редактор с видео, горячими клавишами, автосохранением
- Quality Gate перед экспортом
- Real-time (WebSocket)
- Терминологический модуль
- 6 ролей с RBAC

### Что честно назвать ограничениями

- AI-помощник не реализован
- Чек-лист без UI
- Нет merge сегментов
- Аналитика упрощённая
- Кросстолки — через теги, не полноценная модель

### Слайд 5. Главный сценарий — 5 шагов (1,5 мин)

**Визуал:** линейная цепочка

1. Супервайзер создаёт проект и загружает аудио + JSON
2. Разметчик открывает задачу, правит сегменты
3. Система автосохраняет и проверяет качество
4. Разметчик отправляет на проверку
5. Верификатор принимает → экспорт JSON

**Что сказать:**

> «Это полный цикл из методички: от загрузки до выгрузки. Без ручного «Сохранить» — изменения пишутся сами.»
